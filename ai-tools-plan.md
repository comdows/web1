# 세모플 AI 도구 영역 계획 v1 (2026-07)

> 목표: "내 상황에 필요한 AI가 뭔지 모르겠다"를 푸는 **전 세계 AI 도구 디렉토리 + 상황 기반 추천**.
> 새 화면을 별도로 만들지 않고 기존 디렉토리 인프라(그룹→분야→플랫폼, 검색·비교·즐겨찾기·상세·제보)를
> 그대로 재사용해 6번째 분야 그룹으로 편입한다.

## 1. 구조

- **그룹**: `ai` — 🧠 AI 도구 ("일·콘텐츠·개발을 도와주는 전 세계 AI 도구")
- **분야 10개**: 범용 챗봇 🤖 / 글쓰기·문서 ✍️ / 이미지·디자인 🎨 / 영상 🎬 / 음성·음악 🎙️ /
  개발·코딩 💻 / 회의·기록 📝 / 마케팅·고객응대 📣 / 자동화·에이전트 ⚙️ / 리서치·번역 🔍
- **시드 80개**: 해외 66 + 국내 14 (뤼튼·클로바X·클로바노트·다글로·채널톡·타입캐스트·수퍼톤·라이너·파파고·플리토 등).
  기존 등재 도구 2개(브루·타입캐스트)는 AI 분야로 이동(id·즐겨찾기 유지).
- 전체 디렉토리: 6그룹 · 45분야 · **1,637개**

## 2. AI 도구 찾기 (`?view=ai-finder`)

도구 이름을 몰라도 되는 2단 마법사:
1. **목적 선택** (10개): "AI가 처음", 자료 조사, 글·문서, 이미지, 영상, 음성·음악, 회의록,
   고객 응대·마케팅, 자동화, 개발 — 각 목적이 분야 1~2개로 매핑
2. **상황 선택** (복수): 한국어가 편해요(국내 도구 우선 정렬) / 혼자 운영(최소 조합 권고) /
   코딩 몰라요(노코드 우선 팁) / 쇼핑몰·판매 중(사진→문구→응대 순서 팁)
3. **출력**: 시작 팁 + 분야별 추천 카드(기존 PlatformCard — ★저장·비교·상세 그대로) + 중립 고지

원격 DB에 AI 분야가 아직 시드되지 않았으면 정적 데이터로 자동 폴백.

## 3. 데이터 파이프라인

- 단일 소스: `app/src/data/platforms.json` → `backend/seed/build-seed.mjs`로 0003 재생성(멱등)
- **`backend/migrations/0006_ai_seed.sql`**: AI 부분만 추출한 짧은 시드(그룹·분야·도구 78 insert + 이동 2 update,
  on conflict do nothing — 재실행 안전). ALL.sql도 0006 포함으로 재생성.
- 로컬 Postgres 16으로 ALL.sql 전체 실행 + 0006 재실행(멱등) 검증 완료.

## 4. 운영 원칙

- 중립·사실 위주 소개, 어떤 도구의 대가·제휴비도 받지 않음(디렉토리 비판매 원칙 그대로)
- 요금·상업 사용 조건은 공식 사이트 확인 안내(빠르게 바뀌는 영역이라 세모플이 보증하지 않음)
- 보강 루트: 기존 제보 폼(분야 선택에 AI 분야 자동 노출) → 검수 → 등재
- 확장 여지: 도구별 요금 밴드(fee_band 재사용), "이 일엔 이 조합" 레시피 콘텐츠, AI 분야 비교표 프리셋

## 5. 사용자 실행 필요

Supabase SQL Editor에서 **`backend/migrations/0006_ai_seed.sql`** 실행(1회, 재실행 안전) —
실행 전까지 라이브 사이트의 AI 분야는 비어 보이고(마법사는 정적 폴백으로 동작), 실행 즉시 80개가 표시된다.
