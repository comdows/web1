## 1. 단기 — 로드맵 v2 계승 (Phase 1~6)

순서: **Phase 1 → 3 → 4 연속**, Phase 2·5·6은 외부 트리거 도달 시 인터럽트/병행. 각 페이즈 완료마다 결과 보고 후 다음 착수.

### Phase 1 — 성장 계측 기반 [확정 · 즉시]
- `0034_growth.sql`(events.entity_type/id + metrics_weekly 스냅샷 + admin_snapshot_weekly RPC + 성장/코호트/머니패스 뷰) + metrics.mjs·metrics.yml 주간 cron + admin 성장 패널(시계열 CSS 바·코호트 그리드·머니패스, 차트 라이브러리 금지, "세션 기준 근사" 명기).
- 동승: vite manualChunks(874KB 단일 청크 해소) · 문서 드리프트 정정 · 비AI fee/settle 데이터 소탕.
- **선행 실행 필요**: 미실행 마이그레이션 `0034` + `0036`(헬퍼 grant, 봇 제보 insert 전제) + `0035`(판단 소탕2).

### Phase 2 — 도메인 전환 [트리거: 도메인 구매 · 소 · 인터럽트 가능]
- `scripts/switch-domain.mjs` 1회 + `domain-setup.md` 절차(DNS→Pages HTTPS→Supabase Auth redirect→GSC 신규 속성·주소 변경) → 301·canonical·OAuth 확인. 전환 주를 계측에 라벨링(색인 출렁임 2~6주를 성장 실패로 오판 방지).

### Phase 3 — EN 확충 604→1,719 [트리거: 도메인 확정 직후 / 구매 4주+ 지연 시 즉시 · 대]
- 분야 단위 8~10 PR(EN 엔트리는 name·blurb 2필드 — 방화벽 내장, 빌드 grep 자동 검증) + `en-coverage.mjs`(ko↔en 차집합 리포트) + hub-intros.en 14→45. EN 뉴스 미러는 컷 → 정적 "Recently added"로 대체.

### Phase 4 — KO 편집 콘텐츠 + 재방문 루프 [Phase 3 병행 · 소프트 의존: 계측 4주]
- `articles.ko.json` → `/guide/<slug>/` 정적 페이지(월 2~4편 상한) + KO llms.txt + 홈 개인화 가이드 노출. **진행 중**: 이미 7분야 커버(가이드 4편 완료), 남은 38분야는 계측 데이터로 주제 선정.

### Phase 5 — 유동성 콜드스타트 [트리거: WAU 4주 연속 상승 · Phase 1 패널로 판정]
- 공급 100% 데모 탈출: 등록 퍼널 마찰 제거(머니패스 이탈 실측 후) · 운영자 클레임 승인자 등록 크로스 유도 · outreach 운영 플레이북 문서화 · 데모 전환 기준 문서화. 수집기→제휴 자동 생성은 **부결**(중립·익명 충돌).

### Phase 6 — 수익화 준비 트랙 [상시 병행 · 트리거: 통신판매업 신고 진행]

## 2. 장기 (12~24개월) — 신설 방향

로드맵 v2 너머, 이번 사업·제품 조사에서 도출한 후보. 각 후보에 [트리거]·[선행조건]·[리스크]·불변원칙 부합을 명시.

### L1. 이메일 레이어 2종 활성화 [확정 · Phase 2 종속]
알림 리마인더(0024, 하루 1통 상한·수신거부 구현됨) + 제휴 서버 발송(0015, 이중 게이트·일 10건 상한). **선행**: 발신 도메인 SPF/DKIM/DMARC 인증(github.io로는 불가 → 도메인 구매 필수) · Resend 키 · 처리방침·약관 개정. **도메인 전환(Phase 2)과 한 묶음 진행 권장.** 불변원칙: 정보통신망법 §50 수신거부 준수.

### L2. 비컨헤드(크라우드펀딩) 깊이 증명 [확정 · 계측 트리거]
유형별(리워드/증권/대출/기부) 판단필드 심화 + 전용 비교표 + 가이드 시리즈로 "1개 버티컬 깊이"를 증명. **트리거**: Phase 1 계측으로 크라우드펀딩 검색 수요 확인. **리스크**: 증권형·대출형은 근거법·모금한도가 다른 별개 제도 → **정보 안내만, 접수·소개·자문 없음**(투자중개 경계).

### L3. 운영자 셀프서브 심화 [확정 · Phase 5 종속]
운영자 대시보드에 소식 셀프 등록(검수 유지) · Q&A 답변 통계 · 제안 응답률 지표 추가. **선행**: 운영자 인증 수 증가(Phase 5 유동성). **리스크**: 소개문·정보 직접 수정은 계속 "정정 제안 → 검수" 경유(데이터 품질 방어선 — 의도된 설계 유지).

### L4. 자동등재 점진 개방 [검토]
`autolist`(0016) 고신뢰 임계를 데이터로 상향한 뒤 사후검수 모드로 소량 자동 등재. **트리거**: 검수 큐 신뢰도 축적 데이터. **리스크**: 디렉토리 중립·품질 — 반드시 사후검수 유지, 무검수 등재 금지.

### L5. 계측 기반 버티컬 재우선순위 [확정 · 가정 검증 장치]
Phase 1 머니패스 데이터로 화폐화용 첫 버티컬(현재 가정: 유통·프리랜서)을 실측 재조정. 사업계획 §3의 "비컨헤드↔화폐화 버티컬 분리" 가정을 검증·수정하는 상시 루프.

### L6. 특허 4건 출원 [확정 · 시한 2027-07]
2026-07-03 공개 기준 공지예외 12개월 내 변리사 선행조사·출원. **지연 시 권리 상실.** 우선순위: 발명 1(익명성 자동검증, 구현 완료)·발명 4(밴드 가치진단, 법적 조건).

### L7. 수익화 스위치 온 [트리거 종속]
게이트(G1~G3·X1·X2) + 통신판매업 신고 + 컴플라이언스 체크리스트 완료 후에만. 서버+프론트 이중 게이트 유지. **개발 범위가 아니라 운영 판단** — 30일 전 공지 후 시작.
