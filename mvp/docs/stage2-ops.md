# 2단계(제휴 매칭) 운영 가이드 — 관리자용

> 대상: 플랫폼올 운영자. 접수부터 매칭 성사까지의 표준 절차(SOP), 검수 기준, 지표 정의.
> 3단계(거래소)도 동일 골격으로 운영하되 §5의 추가 원칙을 따른다.

## 1. 접수 채널
- **GitHub Issues** (라벨: `stage2,사전등록` / `stage2,제휴제안` / `stage2,매칭신청`) — 공개 게시판이므로 개인정보 기재 금지(페이지가 안내함). 연락 절차는 이슈 댓글로 안내.
- **이메일** — `data/config.js`의 `contactEmail`에 운영 이메일을 넣으면 페이지의 기본 접수 버튼이 "✉️ 이메일 접수"로 바뀐다(비공개 채널, 권장).

## 2. 검수 체크리스트 (게시 전 전부 통과해야 함)
- [ ] 실재 확인: 공식 URL이 열리고 실제 운영 중인가 (디렉토리 등재 플랫폼이면 `verified: true`)
- [ ] 제안 완결성: 유형 / Give / Get / 규모 밴드가 구체적인가 (막연하면 보완 요청)
- [ ] 금지 업종 아님: 사행성·성인·유사수신 등
- [ ] 개인정보 없음: 연락처·실명·이메일이 본문에 없는가 (빌드 검증이 의심 패턴을 차단하지만 육안 확인)
- [ ] 법 위반 소지 없음: "회원 DB 이관" 류 제안은 개인정보보호법 검토 안내 후 문구 수정
- [ ] 과장·비방 없음: 상대 플랫폼 실명 비교·폄하 문구 제거

## 3. 매칭 SOP
1. **접수** → 이슈에 라벨 확인, 댓글로 접수 확인 + 비공개 연락 절차 안내
2. **검수** (§2) → 통과 시 `data/listings.js`의 `partnerships`에 추가 (`id`는 P-1xx 순번, `posted` 오늘 날짜, `status:"open"`)
3. **배포**: `node build.js` → 커밋·푸시 (빌드가 스키마·개인정보를 자동 검증)
4. **신청 수신** → 신청자 실재 확인 → 등록자에게 신청 요지 전달(비공개)
5. **양측 동의 확인** 후에만 연락처 상호 공유(소개 메일/메시지)
6. **성사 처리**: 해당 리스팅 `status:"matched"`로 변경 → 재빌드·배포. 무산 시 `open` 유지
7. **기록**: 이슈에 결과 코멘트(성사/무산/사유) 남기고 닫기 — 이것이 지표 원천

## 4. 지표 (주간 집계 — GitHub 이슈 라벨 검색으로 센다)
| 지표 | 정의 |
|---|---|
| 사전등록 수 | `label:사전등록` 누적 (오픈 게이트: 기획서 §6.1 "제휴 문의 인바운드") |
| 신규 제안 | 주간 `label:제휴제안` 접수 수 / 게시 승인율 |
| 매칭 신청 | 주간 `label:매칭신청` 수, 리스팅당 평균 |
| 소개 성사 | 연락처 상호 공유까지 간 건수 |
| 제휴 성사 | 양사 "진행 확정" 회신 건수 → 유료화 판단 근거 |

## 5. 원칙 (기획서 §10·§12 v2 — 문구 바꾸지 말 것)
- **자금 미보유**: 정산·결제·에스크로에 관여하지 않는다. 수익화는 성사 건당 정액 중개료(사전 고지)만.
- **개인정보 최소화**: 연락처는 우리 손을 거치는 순간을 최소화하고, 리스팅·이슈 등 공개 지면에 절대 게시하지 않는다.
- **거래소(3단계) 추가 원칙**: 매물은 익명 코드명만 게시, 중개·자문·실사·가치평가를 하지 않는다는 고지를 임의로 완화하지 않는다.

## 6. 오픈 전 로컬 검수 · 활성화 · 요금 전환
- **오픈 전 관리자 열람(비공개)**: 공개 사이트는 "준비 중"만 노출된다. 관리자는 실제 접수 건을
  `data/listings.local.js`(`.gitignore`, 저장소·배포 미포함)에 넣고 **본인 PC에서 localhost로만** 검수한다.
  ```
  cp data/listings.local.example.js data/listings.local.js   # 최초 1회
  cd mvp && python3 -m http.server 8000                       # → http://localhost:8000/partners.html
  ```
  공개 URL에서는 보드가 열리지 않는다(우회 파라미터 없음).
- **정식 오픈(공개 전환)**: `data/config.js` `stage2: true` → 공개용 `data/listings.js`에 게시 승인분만 이동
  (개인정보·비공개 정보 없는지 재확인) → `node build.js` → 커밋·푸시. 그때부터 공개 방문자에게 보드가 보인다.
- 유료 전환 시: partners.html "비용" 섹션 문구 갱신 + 기존 진행 건은 무료 마감 고지
