## 0. 자동등재 사후 검수 🔴 즉시

자동등재는 계획의 "검토/off"와 달리 현재 `enabled:true`, 임계값 80으로 활성화돼 있습니다.
[2026-07-21 수집](https://github.com/comdows/semopl/actions/runs/29874744086)에서 23건이 자동등재됐고, 현재 44건이 사후 검수 대기입니다.

1. 관리 콘솔 → **"🤖 자동 등재 사후 검수"**에서 URL·분야·소개문을 확인해 **확정(검증)** 또는 **내리기**
2. **활성 유지 결정 완료(2026-07-22).** 이후 새 자동등재분을 주간 운영 루틴에서 같은 방식으로 검수
3. 최초 적체 44건을 우선 해소하고 검수 결과를 기록. 임계값 확대는 정확도 근거가 쌓이기 전까지 금지
4. **0043 운영 반영 완료(2026-07-22)** — ad 풀 자동등재 차단의 서버 측 경계 적용됨

## 1. 검색엔진 등록 🔴 최우선

프리렌더 플랫폼 상세 1,719장 + 분야 허브 45장 + EN 플랫폼 1,719장이 이미 배포돼 있는데, 색인 요청을 한 번도
안 해서 검색 유입의 첫 단추가 안 끼워진 상태입니다. 등록만 하면 자산이 일하기 시작합니다.

1. **Google Search Console** — https://search.google.com/search-console
   - 속성 추가 → "URL 접두어" → `https://comdows.github.io/semopl/` (커스텀 도메인 전환 시 domain-setup.md §6 참고)
   - 인증 방식: **HTML 파일** 선택 → 받은 `google*.html` 파일을 이 세션(또는 다음 세션)에
     전달하면 `app/public/`에 넣고 배포해 드립니다 (직접 하려면: 파일을 `app/public/`에 커밋 → Pages 배포 후 "확인")
   - 인증 후: Sitemaps 메뉴 → `sitemap.xml` 제출
2. **Bing 웹마스터 도구** — https://www.bing.com/webmasters
   - "GSC에서 가져오기"를 지원해 1이 끝나면 클릭 몇 번으로 끝. Bing은 ChatGPT 검색의 소스라 중요.
3. **네이버 서치어드바이저** — https://searchadvisor.naver.com
   - 사이트 등록 → HTML 파일 인증(`naver*.html` — 1과 동일하게 전달 주시면 반영) → 사이트맵 제출
4. 등록 1~2주 뒤부터 GSC "실적" 리포트 확인 — 관리 콘솔 "퍼널·유입" 패널(0017)과 교차 검증.

## 2. 알림 이메일 켜기 (선택 — 리텐션 루프 완성)

코드·DB·수신거부 페이지(`?view=optout`)까지 준비 완료, **스위치만 꺼져 있음**.
절차는 README "알림 이메일 켜기" ①~⑤ 그대로:
Resend 가입 → 발신 도메인 SPF/DKIM/DMARC 인증 → `supabase functions deploy send-notify-email`
+ secrets(RESEND_API_KEY·NOTIFY_EMAIL_FROM·CRON_SECRET) → 처리방침에 알림 메일 항목 반영 →
`app_settings 'notify_email'` enabled:true → 스케줄(Supabase Cron 또는 notify.yml curl 스텝).
제휴 제안 서버 발송(send-proposal)도 같은 인프라를 쓰므로 이때 함께 켜는 것이 효율적입니다.

## 5.5 배포 후 5분 스모크 (큰 배포·마이그레이션 직후 1회)

자동 검증(CI·스모크 하네스)이 못 보는 **실환경 연결**만 확인합니다. 순서대로 5분:

1. **홈** — `https://comdows.github.io/semopl/` 새로고침: 히어로·분야 그리드·"이번 주 많이 찾은" 렌더,
   플랫폼 수가 0이 아님(원격 로드 성공 신호). 흰 화면이면 즉시 이전 배포로 롤백 검토.
2. **검색·상세** — 아무 검색어 입력 → 결과 → 카드 클릭 → 상세 렌더 + ☆ 즐겨찾기 토글.
3. **로그인 왕복** — 로그인 → 홈에 개인화 섹션 표시 → 계정 페이지 진입 → 로그아웃.
   (마이그레이션 직후라면: 제보 폼 1건 제출 → 관리 콘솔 검수 큐에 뜨는지까지.)
4. **관리 콘솔** — `?view=admin`: 상단 지표 숫자 정상(0/—가 아닌 실값), **운영 잡 헬스** 카드에
   ❌/⚠️ 없는지, **프론트 오류** 카드에 새 오류가 쌓이지 않는지(배포 직후 몇 분 관찰).
5. **정적 경로** — 프리렌더 상세 1곳(`/semopl/p/coupang/`)과 EN 랜딩(`/semopl/en/`) 직접 URL 진입 —
   200 + 제목 정상(프리렌더 회귀 확인).

이상 발견 시: 프론트 오류 카드의 메시지 확인 → 해당 뷰·마이그레이션 순으로 원인 좁히기.
(참고: 로컬 전수 검증은 `backend/test/README.md` — 라우트 37체크·적대 6체크·PG16 시나리오.)
