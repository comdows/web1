## 1. Executive Summary

**문제.** 사업자·개인이 "나가서 붙을 플랫폼"(판매채널·입점처·자금조달처·매칭처)을 고를 때,
정보가 블로그·유튜브·카페에 파편화돼 탐색 비용이 크다. Capterra·G2는 *내부에서 쓰는
소프트웨어*만 다루고, *외부에 붙는 플랫폼*은 아무도 구조적으로 분류·비교하지 않는다.

**해법(3단계).**
- **1단계(Wedge) 발견·비교·평가:** 플랫폼을 구조적으로 분류하고 수수료·정산주기·입점조건·정산리스크를 표준화 비교. 수요측 트래픽을 SEO로 확보.
- **2단계 제휴 매칭(BD as a Service):** 플랫폼 간 가입자 교환·송출·정산·협업 중개.
- **3단계 M&A·엑시트 거래소:** 동종 플랫폼 매매·엑시트 수요 연결.

**타깃(1단계).** 1차: 플랫폼을 고르려는 사업자/개인(수요). 2차(데이터 소스): 플랫폼 운영자.

**차별점/해자.** ① 아무도 표준화하지 않은 "외부 부착형 플랫폼" 비교 스키마, ② 이음의 다국어·프로그래매틱 SEO 인프라로 만든 롱테일 방어막, ③ 1단계 트래픽이 2·3단계 양면 유동성 씨앗이 되는 구조적 연결.

**North Star.** *주간 "의사결정 도달 비교 세션 수"*(≥2개 플랫폼 비교 + 상세/외부이동/계산 실행).

**최대 리스크.** 콜드스타트가 아니라 **폭의 유혹.** 6개 버티컬을 동시에 얕게 채우면 비교 깊이가 없어 검색·재방문 모두 실패. → 비컨헤드 1개로 깊이를 먼저 증명.

## 2. 배경 & 문제 정의

### 2.1 왜 지금인가
- **플랫폼 과잉:** 판매채널·프리랜서마켓·크라우드펀딩·공동구매·수출입·배달 등 "붙을 수 있는" 채널 폭증 → 선택 피로 증가.
- **정보 비대칭 고착:** 각 플랫폼은 유리한 정보만 노출(수수료 하한만, 정산주기 애매, 페널티·정산유보 미고지). 중립적 3자 비교의 부재가 구조적으로 지속.
- **AI 검색 전환기:** 생성형 검색이 파편 콘텐츠를 요약 → "구조화된 1차 데이터"를 가진 쪽이 인용·유입 흡수. 지금 스키마를 선점하지 않으면 남의 요약 소스로 전락.

### 2.2 누구의 어떤 페인인가

| 주체 | Job(페인) | 현재 대체행동 | 공백 |
|---|---|---|---|
| **이용 사업자(수요, 1차)** | "내 상황에 맞는 판매/조달/자금 채널을 최소 리스크로 고르고 싶다" | 블로그·유튜브·카페·지인 | 비교축 제각각, 수수료·정산 실체 불투명, 최신성 낮음 |
| **개인(수요)** | "프리랜서마켓/펀딩 어디에 올릴지 고르고 싶다" | 커뮤니티 검색 | 광고성 후기, 정산·페널티 정보 부재 |
| **플랫폼 운영자(공급, 2차)** | "우리 채널에 맞는 셀러/제휴처를 저비용으로 확보하고 싶다" | 직접 영업, 광고 | 발견 채널 부재, BD 인건비 과다 |

> **정량화의 정직한 한계.** 위 페인의 국내 공개 통계는 빈약하다. 따라서 초기엔 **검색 수요(키워드 검색량)**를 관측 가능한 프록시로 사용한다(예: "쿠팡 수수료", "스마트스토어 정산주기", "와디즈 수수료" 롱테일 합). 실제 수치는 런칭 전 키워드 리서치로 확정 — 지금 숫자를 지어내지 않는다.

## 3. 시장 분석 & 경쟁 지형 (3개 레이어)

### 3.1 레이어별 수요·대체재·공백

| 레이어 | 수요 실재성 | 주요 대체재 | 공백(우리 자리) | 화폐화 난이도 |
|---|---|---|---|---|
| **1. 발견·비교** | 높음(검색 관측) | 블로그/유튜브 · G2·Capterra(내부SW만) · 커머스 한정 디렉토리 | 외부 부착형 플랫폼 **범용 비교 스키마 부재** | 낮음(광고/입점) — **먼저** |
| **2. 제휴 매칭** | 중간(운영자 페인 크나 발견 채널 없음) | 직접영업·제휴대행·DM | 상호 송출·정산 **표준 레일 부재** | 중간(정산 수수료) |
| **3. M&A 거래소** | 낮음·희소(고가치·저빈도) | M&A 부티크·사모·개인 브로커·매물 카페 | 소형 플랫폼 특화 거래소 부재 | 높음(거래 수수료·법무) |

### 3.2 경쟁 포지셔닝 맵

```
                내부에서 쓰는 SW ───────────── 외부에 붙는 플랫폼
                (CRM/ERP/회계)                (판매채널/조달/자금)
   범용·구조화   ┌──────────────┐            ┌──────────────┐
   (비교 스키마) │  G2/Capterra │            │   ★ 우리     │  ← 빈 사분면
                └──────────────┘            └──────────────┘
   파편·비구조   ┌──────────────┐            ┌──────────────┐
   (블로그성)    │  IT 리뷰블로그│            │ 커머스채널 후기│
                └──────────────┘            └──────────────┘
```
- **핵심 통찰:** 우측 상단(외부 부착형 × 구조화)이 비어 있다. 단 "넓게 얕게" 채우면 즉시 우측 하단(블로그성)으로 추락. **구조화(깊이)가 곧 진입장벽.**

### 3.3 "폭넓은 버티컬" 결정의 냉정한 평가
- **장점:** SEO 표면적이 넓어 롱테일 유입 총량이 크고, 이음 프로그래매틱 SEO 자산과 궁합 좋음.
- **치명 위험:** 버티컬마다 비교축이 다르다(프리랜서마켓의 "수수료·노출알고리즘" ≠ 수출입의 "관세·통관·에스크로" ≠ 공동구매의 "MOQ·정산유보"). **6개를 동시에 스키마화할 인력 없음**(6~8명).
- **결론:** 폭은 **카탈로그(얕은 목록·기본정보)** 레벨로만 즉시 수용, **비교 깊이**는 **비컨헤드 1개**에 먼저 투입.

## 5. 핵심 가치 제안 & 차별화(해자)

### 5.1 가치 제안
- **수요측:** "흩어진 플랫폼 정보를 **같은 자로 재서** 3분 만에 비교"(특히 숨은 비용·정산리스크 표준화).
- **공급측:** "우리 플랫폼을 정확·유리하게 노출하고, 나중엔 맞는 파트너까지 붙여준다."

### 5.2 방어 가능한 해자 (진입 순서로 두꺼워짐)

| 해자 | 원천 | 모방 난이도 |
|---|---|---|
| 비교 스키마 IP | 버티컬별 정규화 속성·정산리스크 사전 | 높음(도메인 노가다·검증) |
| 다국어 프로그래매틱 SEO | 이음 30개국어·SEO 인프라 재활용 | 높음(인프라·시간) |
| 1차 데이터 신뢰 | 운영자 클레임 + 사용자 제보 이중검증 | 중간(운영 리소스) |
| 양면 유동성(1→2 전환) | 수요 트래픽이 공급 광고·제휴 유인 | 높음(선점·네트워크) |

### 5.3 이음 자산 레버리지
- **SEO 인프라:** 버티컬×플랫폼×속성 조합으로 프로그래매틱 페이지 대량 생성 → 롱테일 흡수.
- **다국어:** 수출입·크라우드펀딩 등 국경 넘는 버티컬에서 30개국어가 즉시 차별점.
- **지역×산업 회원 데이터:** 1단계 기여는 적음(솔직히). 단 2단계 "같은 단지/산업 내 매칭"으로 재활성화 가능.

## 6. 서비스 전략 & 단계별 로드맵

### 6.1 확장 논리
```
[1단계 발견·비교] ──트래픽/데이터──▶ [2단계 제휴 매칭] ──신뢰/정산레일──▶ [3단계 M&A 거래소]
```
확장은 자동이 아니다. 명시적 **게이트**를 둔다.

| 전환 | 게이트(가정치, 후 보정) | 미달 시 |
|---|---|---|
| 1→2 | 비컨헤드 MAU ≥ 3만, 유료 입점 ≥ 30곳, "제휴 문의" 인바운드 관측 | 1단계 깊이·광고 지속 투자 |
| 2→3 | 제휴 정산 GMV 누적 검증, 매물 인바운드 반복, 법무 자문 확보 | 2단계 확장/추가 버티컬 |

### 6.2 "넓은 카탈로그 + 좁은 비컨헤드" 이중구조 (핵심 베팅)
```
카탈로그(넓게):  프리랜서·크라우드펀딩·배달·수출입·공동구매·유통  ← 얕은 기본정보 + SEO
                       │
                       ▼ (그중 1개만 깊게)
비교(깊게):      ★ 비컨헤드 1개 ★  ← 정규화 속성·정산리스크·검증데이터·에디토리얼·계산기
```
깊이가 증명되면 그 템플릿을 다음 버티컬로 복제.

## 7. 플랫폼 분류 체계(Taxonomy) & 정보구조

### 7.1 분류 축 (다차원 태깅)
1. **부착 목적:** 판매채널 / 자금조달 / 인력·용역 / 조달·구매 / 물류·풀필먼트 / 마케팅·노출
2. **버티컬:** 프리랜서마켓 · 크라우드펀딩 · 배달 · 수출입 · 공동구매 · 유통 …
3. **정산 모델:** 수수료율 / 정산주기 / 정산유보·에스크로 / 광고강요 여부
4. **입점 조건:** 사업자유형 / 카테고리 제한 / 심사기간 / 초기비용
5. **규모·타깃:** 국내/해외, B2B/B2C, 거래규모대
6. **지역(이음 자산):** 단지·권역 태그(2단계 매칭용)

### 7.2 비교 속성 & 데이터 스키마 (여기서 승부)

**공통 스키마:**
```json
{
  "platform_id": "string",
  "name": "string",
  "vertical": ["string"],
  "purpose": ["string"],
  "region": ["string"],
  "languages": ["string"],
  "fee_model": { "type": "percent|flat|mixed", "rate_min": 0, "rate_max": 0, "hidden_fees": [] },
  "settlement": { "cycle_days": 0, "holdback_pct": 0, "escrow": false, "dispute_policy": "string" },
  "onboarding": { "biz_type_required": "string", "categories_allowed": [], "review_days": 0, "setup_cost": 0 },
  "scale": { "b2b_b2c": "string", "geo": "string", "traffic_tier": "관측|추정(근거표기)" },
  "evidence": { "source": "운영자|공식약관|사용자제보", "verified_at": "date", "confidence": "high|mid|low" }
}
```
**버티컬 확장 필드(예):**
- 크라우드펀딩: `success_fee_pct`, `pg_fee`, `refund_policy`, `funding_type(리워드/투자)`
- 수출입: `incoterms_지원`, `통관대행`, `결제보증(TT/LC/에스크로)`, `대상국`
- 프리랜서마켓: `노출알고리즘`, `등급제`, `구매자수수료 별도`

> **데이터 무결성 원칙:** 모든 수치에 `evidence.source`·`verified_at` 강제. "근거 없는 수수료 표기"는 스키마 레벨에서 금지 → 이것이 블로그성 콘텐츠와의 신뢰 격차.

### 7.3 비컨헤드 선정 기준
`(검색 수요) × (정산리스크 정보 비대칭=유틸리티) × (이음 자산 적합도) ÷ (데이터 획득 난이도)`
- 유력 후보(런칭 전 검증 확정): **크라우드펀딩**(수수료·성공수수료·PG·환불 복잡, 플랫폼 수 유한→획득 쉬움) 또는 **수출입**(다국어 자산 강함, 정보 비대칭 극심, 단 스키마 복잡).

## 8. MVP 범위 & 핵심 기능 정의 (1단계)

### 8.1 넣는 것 / 빼는 것

| 구분 | 넣는다(Must) | 뺀다(Not now) |
|---|---|---|
| 콘텐츠 | 비컨헤드 1개 깊은 비교 + 나머지 5개 얕은 카탈로그 | 6개 전부 깊은 비교 |
| 비교 | 2~4개 나란히 비교 테이블 + 정산리스크 하이라이트 | 개인화 추천엔진/AI 상담 |
| 탐색 | 버티컬·목적·조건 필터, 프로그래매틱 SEO 랜딩 | 로그인 필수 기능 |
| 신뢰 | evidence 표기, 오류 제보 버튼 | 풀 UGC 리뷰(어뷰징 부담) |
| 유틸리티 | 수수료·실수령 계산기(정산유보 반영) | 계정연동/실거래 트래킹 |
| 화폐화 | 입점/상위노출 광고 슬롯(수동 판매) 자리 확보 | 셀프서브 광고입찰 |
| 공급측 | 운영자 정보정정 클레임 폼(무료) | 운영자 대시보드 |

### 8.2 핵심 화면
1. **버티컬 허브(프로그래매틱):** `/{vertical}/플랫폼-비교` — SEO 랜딩, 목록·필터.
2. **비교 테이블:** 2~4개 선택 → 같은 축, 숨은비용·정산리스크 배지.
3. **플랫폼 상세:** 스키마 전 항목 + evidence + 제보 버튼 + (광고주) 강조 슬롯.
4. **계산기(단면 유틸):** 매출/모금액 입력 → 수수료·정산유보·실수령·정산일. **로그인 불필요.**
5. **운영자 정정 폼:** 데이터 오류 신고(공급측 첫 접점).

### 8.3 단면(single-player) 유틸리티 = 콜드스타트 완화
- 계산기·비교 테이블은 **다른 사용자 없이도 즉시 유용** → 네트워크 없이 첫 가치 제공.

## 9. 콜드스타트 전략 (정면 대응)

### 9.1 문제 정의(정직하게)
- 1단계의 진짜 콜드스타트는 "광고주"가 아니라 **콘텐츠(데이터) 공급**이다. 데이터는 **우리가 직접 시딩** 가능하므로 닭-달걀이 아니라 **노동 문제** — 이게 이 서비스의 행운.

### 9.2 공급(데이터) 확보 — "우리가 만든다"
1. **에디토리얼 시딩:** 비컨헤드 1개를 손으로 완전 스키마화(플랫폼 15~40개), 공개약관·요금표·운영자 확인.
2. **프로그래매틱 확장:** 나머지 5개는 공개정보 기반 얕은 카탈로그를 SEO 인프라로 대량 생성.
3. **운영자 정정 루프:** 정정 폼으로 운영자가 스스로 품질 향상(무료·상호이익).
4. **다국어 자산:** 해외 플랫폼도 30개국어 커버 → 데이터 폭 저비용 확장.

### 9.3 수요(트래픽) 확보 — SEO 우선
1. **프로그래매틱 SEO 주엔진:** "○○ 수수료/정산주기/vs" 롱테일을 버티컬×플랫폼×속성 조합으로 대량 랜딩.
2. **비교·계산기의 검색 친화성:** "실수령 계산" 류는 검색·공유·백링크가 잘 붙음.
3. **커뮤니티 시딩:** 셀러·창작자 카페/오픈채팅에 계산기·비교표를 답변형으로 배포(스팸 아닌 유틸 제공).
4. **AI 검색 대비:** 구조화 데이터(스키마 마크업)로 생성형 검색 인용 소스화.

### 9.4 왜 "폭"이 콜드스타트를 위협하는가 — 완화책
- 위험: 6개 얕은 버티컬 = 검색 진입해도 "블로그보다 나을 게 없음" → 이탈·재방문 실패 → SEO 신호 악화.
- 완화: 비컨헤드에서 "우리만의 깊이"를 먼저 증명(체류·비교완료·재방문↑) → 검증 후 템플릿 복제. 얕은 5개는 "존재하되 기대치 관리".

### 9.5 광고 화폐화의 콜드스타트
- 광고는 트래픽 선행 필요 → MVP엔 슬롯 자리만, 비컨헤드 MAU 게이트(수만) 넘으면 수동 세일즈로 첫 광고주 확보. 셀프서브는 나중.

## 10. 수익 모델 & 유닛 이코노믹스

### 10.1 레이어별 수익원 (우선순위: 광고 먼저)

| 순위 | 레이어 | 수익원 | 과금 구조 | 전제 |
|---|---|---|---|---|
| **1** | 1단계 | 입점/상위노출 광고 | 월정액 슬롯 + 프리미엄 프로필 | 트래픽 선행 |
| 1b | 1단계 | 리드/아웃바운드 클릭 | CPC 또는 CPL | 볼륨 필요 |
| 2 | 2단계 | 제휴 정산 수수료 | 송출→구매 GMV의 % (take-rate) | 양면 유동성 |
| 3 | 3단계 | 거래 수수료 | 성사 시 %(성공보수) | 신뢰·법무 |

### 10.2 개략 유닛 이코노믹스 (가정 명시 — 실측 대체 골격)
- **1단계 광고(가정):** 유료 입점 플랫폼당 월 30~100만 원. 데이터 신뢰가 CPM 프리미엄 유발.
  - 손익분기(대략): 월 고정비 ≈ 4~6천만 원 → **유료 입점 60~150곳** 필요(월 40만 원 가정).
  - → **1단계 단독 흑자는 어렵다.** 1단계는 **트래픽·데이터 자산 축적기**로 보고, 흑자는 2단계 take-rate에서 노림(정직한 평가).
- **2단계 제휴(가정):** GMV take-rate 3~10%. 소수 대형 제휴가 롱테일 광고보다 매출 기여 클 가능성.
- **CAC:** SEO 주도라 유료 CAC 낮게 설계(콘텐츠·인프라 고정비로 치환). **핵심 비용은 마케팅이 아니라 데이터 큐레이션 인건비.**

> **냉정한 결론:** 광고는 "첫 현금"이지 "사업의 엔진"이 아니다. 광고만으로는 팀을 못 먹인다. 광고는 2단계 제휴 매출로 가는 다리로 취급하고, KPI도 광고매출보다 "트래픽·데이터 신뢰·제휴 인바운드"를 앞에 둔다.

## 11. 성공 지표 (North Star + KPI)

**North Star:** 주간 *"의사결정 도달 비교 세션"*(≥2개 플랫폼 비교 + 상세/외부이동/계산 실행).
이유: PV(허수)가 아니라 "고르는 일을 실제로 도왔는가"를 측정.

| 단계 | 선행지표 | 후행지표 |
|---|---|---|
| 데이터 시딩 | 스키마화 플랫폼 수, evidence 검증율 | 데이터 신뢰(정정건수↓) |
| SEO/수요 | 색인 페이지 수, 롱테일 노출·CTR | 비컨헤드 MAU, 재방문율 |
| 유틸리티 | 계산기 사용수, 비교완료율 | 세션당 비교 수, 체류시간 |
| 화폐화 | 정정폼 제출수, 광고 문의 | 유료 입점 수, 광고 MRR |
| 1→2 전환 | 제휴 문의 인바운드 | 제휴 성사·정산 GMV |

**검증 게이트(kill 기준):**
- 3개월: 비컨헤드 비교완료율/재방문이 블로그 대비 유의미하지 않으면 → 깊이 재설계(폭 확장 금지).
- 6개월: 유료 입점/광고 문의 0건 → 화폐화 가설 수정.

## 부록 A. 한 장 요약 (의사결정용)
- **먼저:** 수요측 트래픽을, 비컨헤드 1개의 깊은 비교로. (폭은 카탈로그로만.)
- **엔진:** 이음의 다국어·프로그래매틱 SEO + evidence 기반 신뢰.
- **첫 현금:** 입점/상위노출 광고(단, 사업 엔진은 2단계 제휴 take-rate).
- **최대 리스크:** 폭의 유혹 → 깊이 실패. 게이트로 확장 통제.
- **kill 기준:** 3개월 내 비컨헤드가 블로그를 못 이기면 폭 확장 금지, 깊이부터 다시.
