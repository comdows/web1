# 세모플 기능 고도화 로드맵 (우선순위)

작성: 2026-07-11 · 기준: 실사용자 유입 직전(0001~0029 배포 완료, 도메인 전환 준비 완료, SEO 자산 구축).

## 판단 원칙
세모플은 기능 표면이 이미 촘촘하다(22개 뷰, 수익화 6종·이메일·자동수집이 **코드 완성 + 스위치 off**). 따라서 "새 기능"보다 **이미 만든 자산을 재방문·전환으로 연결**하는 고도화가 레버리지가 크다. 우선순위는 임팩트(리텐션·전환·유입) × 실행난이도 × 기존 인프라 재사용도로 매긴다.
불변 원칙 유지: 디렉토리 중립(유료·광고 개입 없음)·연락처 이중 차단·공개 뷰 익명성·성공보수 금지·service_role 금지·EN 방화벽·한국어 UI.

## P0 — 유입을 재방문으로 (지금)

### H-1. 저장된 검색 + 조건 매칭 알림  ⭐ 최우선
- **공백**: 검색조건(분야·지역·수수료대·키워드)을 저장·구독하는 기능이 전무. B2B 사용자가 "이 분야에 새 플랫폼 뜨면 알려줘"를 못 함(saved_search grep 0건). notify.mjs는 즐겨찾기 분야만 암묵 신호로 사용.
- **왜 1순위**: 일회성 조회를 모니터링 도구로 전환하는 리텐션 핵심 루프. `notifications`(0018)·`notify.mjs`·인앱 알림 UI가 이미 있어 확장 비용 최소.
- **범위**: 0030 `saved_searches`(user_id·라벨·조건 jsonb·last_notified, RLS 본인+상한) → notify.mjs 확장(신규 등재 플랫폼이 저장조건 매칭 시 `search_match` 알림, 멱등) → 검색 페이지 "이 조건 저장·알림받기" 버튼 + 계정 "내 저장 검색" 관리(store 조건 직렬화 재사용).
- **규모**: 마이그레이션 1 + 프론트(discovery·account) + notify.mjs. 중.
- **검증**: PG16 RLS(본인만·상한), notify --dry 픽스처(저장조건×신규 매칭), Playwright(저장→목록→알림 mock).

### H-2. 로그인 개인화 — 서버 관심 프로필 + 개인화 홈
- **공백**: 홈이 익명·로그인 동일(App.tsx). 관심·최근·즐겨찾기가 전부 localStorage → 기기 간 단절(즐겨찾기만 서버 동기화). 서버 관심 프로필이 없어 알림·추천 품질이 기기 종속.
- **왜**: H-1과 결합해 개인화 알림·추천 품질을 끌어올림. 로그인 사용자에게 "돌아올 이유"를 홈에서 제시.
- **범위**: onboarding 관심(그룹·분야·신규선호)을 서버로(profiles 컬럼 또는 `user_interests`, favsync.ts 패턴 재사용) → 로그인 홈 상단 개인화 블록(저장 검색 요약·관심 분야 신착·즐겨찾기 소식). 익명은 현행 유지.
- **규모**: 마이그레이션 1(경량) + 프론트(App 홈 분기·store 동기화). 중.
- **검증**: 기기 간 동기화 시나리오, Playwright(로그인 홈 개인화 블록 렌더·익명 무변화).

## P1 — 품질·깊이

### H-3. AI 도구 심화 — 전용 비교축 + 필터
- **공백**: AI 163개는 fee_band/fee_text/settle_text가 전부 0%. AI끼리 비교·필터가 사실상 무의미(설명·분야만).
- **범위**: AI 전용 축 도입 — 무료여부·요금형태(무료/프리미엄/구독/종량)·한국어지원 등(정적 데이터 필드 + 0031 시드, 0029 패턴). AI 상세·비교·ai-finder에 축 반영. 디렉토리 중립 유지(요금 표기는 "형태"만, 가격 금액 게재 금지).
- **규모**: 데이터 작성(163개) + 마이그레이션 + 프론트. 중.

### H-4. 매칭 지능화 (Track B — 코드 내 미완 명시)
- **공백**: 브리프↔매물 실제 게이트가 분야+형태뿐. 예산은 점수만(match.ts budgetFit)·지역은 어디서도 미반영. notify.mjs:24가 "예산·지역은 Track B에서"라 자인.
- **범위**: `matches()`/`briefMatchesDeal`에 예산 밴드 근접·지역 일치 반영, scoreBriefDeal을 알림 게이트에도 연결. 제휴 scorePartnerFit에 규모 밴드 추가(가격중립 유지).
- **규모**: match.ts + notify.mjs + api.ts. 중.

## 실행 방식
배치마다: 리셋→구현→PG16(해당 시)→Playwright→커밋→PR→CI→squash→Pages. 마이그레이션은 멱등, ALL.sql 재생성, README 실행 체크박스. 각 배치 배포 후 사용자에게 SQL 실행 안내(해당 시).
권장 순서: **H-1 → H-2**(P0 리텐션 팩) → H-3/H-4(품질) → H-5(콘텐츠). 수익화 점등은 이용량·법적 준비가 트리거.
