# 세모플 2단계 수익화 기획안 (v1)

> 작성: 전문가 3팀(법률·규제 / 과금 벤치마크 / 마켓플레이스 운영) 병렬 분석 결과를 종합·반영.
> 전제: **자금 미보유 원칙 유지** — 세모플은 제휴 대금·정산에 일절 관여하지 않고, 자기 서비스 이용료만 받는다.

---

## 0. 세 팀이 수렴한 결론 (한눈에)

1. **"거래액 % 성사 수수료"는 버린다.**
   - 법률팀: 4개 과금 구조 중 최고 리스크(정산액 파악 필요 → 자금 흐름 개입 유혹 → 전자금융 규제 전환 위험).
   - 벤치마크팀: 크몽·위시켓처럼 거래가 플랫폼 안에서 정산될 때만 성립하는 모델. 세모플은 거래를 관측할 수 없음.
   - 운영팀: 관측 불가능한 "성사"를 과금 단위로 삼으면 회피(바이패스)가 구조적으로 100% 가능.
2. **과금 단위는 "성사"가 아니라 "소개 실행(연락처 상호 공유)"이다.**
   - 세모플이 자기 눈으로 100% 확인 가능한 유일한 이벤트. 과금이 소개보다 앞서므로 회피가 원천 봉쇄된다.
   - 기존 기획서의 "성사 건당 정액 중개료" 표현을 **"연결(소개) 건당 정액 이용료"로 교정**한다.
3. **수익은 3층 하이브리드로, 시차를 두고 쌓는다**: ① 스폰서 노출(광고) → ② 연결료(소개 정액) → ③ 구독 멤버십.

## 1. 수익 모델 — 3층 하이브리드

### T1. 스폰서 노출 (첫 수익 — 유동성 게이트 1 통과 시)
- **무엇**: 제휴 매칭 보드·제휴 카탈로그 내 상단 고정/추천 슬롯. 월 정액.
- **가격 가이드**: 월 5만~15만 원 (직방 일반매물 5만~16.5만 벤치마크).
- **필수 조건**: `AD/스폰서` 라벨 명시(표시광고법) + **디렉토리(비교·순위·검색) 지면에는 절대 적용 금지** — 노출 판매는 매칭 보드 한정.

### T2. 연결료 (핵심 수익 — 유동성 게이트 2 통과 시)
- **무엇**: 매칭 신청 → 검수 → **양측 동의 확인 → 연락처 상호 공유(소개 실행)** 순간에 정액 과금. 신청·매칭 확인까지는 무료.
- **가격 가이드** (제휴 방식 유형별 차등 — 카탈로그 22종의 3계층):
  | 유형 | 예 | 연결료 |
  |---|---|---|
  | A. 무정산·저마찰 (settlement=none) | 배너 맞교환·뉴스레터 스왑·쿠폰 교환 | **무료~1만 원** (유동성 미끼) |
  | B. 성과 레퍼럴 (settlement=direct) | 회원 상호송출·레퍼럴·어필리에이트 | **1만~3만 원/건** (숨고 리드 4천~6천 원 상회 수준) |
  | C. 깊은 연동 (effort=heavy) | API 연동·데이터 제휴·버티컬 연합 | **5만~10만 원/건** |
- **소액 원칙**: "회피할 마음이 안 들 만큼" — 성과형 고액을 노리는 순간 회피율이 폭증한다(운영팀). 저가·정액이 총수익을 지킨다.
- **선불 크레딧 지갑**: 연결료는 크레딧 차감 방식(충전 시 소량 보너스). 무료 티어에 월 무료 크레딧 소량 지급.
- **성사(제휴 체결)는 과금하지 않는다** — 대신 **자진신고 시 보상**(파트너 배지·보드 상위노출·다음 연결 할인). 성사 데이터는 과금이 아니라 신뢰자산·요금 조정 근거로만 사용.

### T3. 구독 멤버십 (12개월+ — 네트워크·데이터 가치 축적 후)
- **Free**: 등재·제휴 프로필·A형 무제한·월 무료 크레딧 1~2건.
- **Pro (월 5만~9만 원)**: 연결 크레딧 포함·파트너 검색 무제한·검증 배지·트래킹 대시보드.
- **Premium (월 20만~30만 원)**: 매칭 매니저 큐레이션·C형 우선 소개·제휴 계약서 템플릿·성과 리포트.
- 링크드인($99+/월)·알리바바(연 $2,000+)·이랜서(6개월 30만 원) 벤치마크 범위 내.

### (3단계 거래소 과금 — 별도 원칙)
- **리스팅료/정보이용료(정액)만**. 성사·거래금액과 절대 비연동(자본시장법 "중개" 논란 회피).
- "중개·자문·실사 아님" 고지를 과금 구조·약관·마케팅 전반에서 일관 유지. 익명 리스팅·당사자 직접 접촉 불변.

## 2. 유료화 타이밍 — 유동성 게이트 (수치 기준)

**게이트 통과 전에는 전면 무료.** 지표는 GitHub 이슈 라벨/보드 데이터로 집계(운영 부하 0).

| 게이트 | 조건 (4주 연속 유지) | 열리는 과금 |
|---|---|---|
| **G0 (지금)** | — | 전면 무료. 세모플이 직접 리스팅을 채워 공급 확보 |
| **G1** | 활성 리스팅 ≥ 30 · 월 인바운드 문의 ≥ 10 | **T1 스폰서 노출** |
| **G2** | 활성 리스팅 ≥ 50 · 리스팅당 신청 ≥ 3 · 누적 소개 ≥ 20 · 소개 성사율 ≥ 25% | **T2 연결료** (기존 진행 건은 무료 마감·그랜드파더링) |
| **G3** | 월 연결 ≥ 30 · 재사용 고객 ≥ 30% | **T3 구독** 전환 |

## 3. 리스크 검토 결과와 반영 사항

### 법률·규제 (스크리닝 — 확정 전 변호사·세무 검토 필수)
| 리스크 | 반영 |
|---|---|
| **자금 흐름 개입 → 전자금융업 규제** | 레드라인: 세모플 계좌에 제휴·M&A 대금이 스치는 것 자체를 금지. 세모플은 자기 명의 이용료만 수취 |
| 통신판매중개자 책임(전자상거래법 §20) | "세모플은 제휴의 당사자가 아님" 고지 문구 선제 적용 + 통신판매업 신고(방어적) |
| 포괄 면책약관 무효(약관규제법 §7) | 면책은 "알선·소개에 한정, 이행·정산은 당사자 책임" 등 사실 부합 범위로만. "성사" 정의를 객관 기준으로 약관 명문화 |
| 유료 노출 미표기(표시광고법) | 모든 유료 노출에 AD 라벨 + 디렉토리 지면 분리 |
| 리드 소개 시 개인정보 제3자 제공 | 개인 리드는 정보주체 별도 동의 필수. 원칙적으로 개인정보는 두 플랫폼이 직접 주고받게 하고 세모플 처리 최소화 |
| 성과·수익 보장 마케팅 | 전면 금지("제휴 성과 보장" 류 문구 사용 불가) |
| M&A "중개" 표현 | 금지. 리스팅·연결 서비스로만 표기, 정액 과금만 |

### 운영 (유료화 7원칙 — 전문 별첨 분석)
1. 과금 단위 = 소개 실행(연락처 공유), 성사 아님
2. 과금 시점은 소개보다 앞선다(회피 원천 봉쇄)
3. 연결료는 소액 정액(회피 유인 제거)
4. 유동성 게이트 전 무료, 첫 유료화는 "노출"부터
5. **"우리는 소개를 팝니다, 성사를 팔지 않습니다"** — 환불은 세모플의 소개 미이행(상대 무응답 N일 초과 등)에만, 협상 결렬엔 불가. SLA 부재는 지표 투명 공개로 대체
6. 돈·세금·환불은 100% 자동화(PG 결제링크 + 세금계산서 API 자동발행 + 크레딧 지갑) 후에만 유료화 시작. 판단(검수)·신뢰(소개)만 사람
7. **중립성 벽**: 디렉토리 정보(비교·순위·검색)는 판매 불가. 판매 대상은 오직 연결(매칭 보드). 성사 배지도 디렉토리 순위 반영 금지

### 컴플라이언스 체크리스트 (유료화 개시 전)
- [ ] 사업자등록 + 통신판매업 신고
- [ ] 이용약관: 성사 정의·환불 기준·면책 범위·우회 무산신고 금지 조항
- [ ] 세금계산서 자동발행(팝빌 등 API) + VAT 10% 포함 가격 표기
- [ ] AD 라벨 컴포넌트 + 디렉토리/보드 지면 분리 확인
- [ ] 개인정보 처리방침 갱신(리드 소개 국면 3자 제공 동의 플로우)
- [ ] 변호사·세무사 최종 검토

## 4. 손익 감각 (보수적 시나리오, G2 통과 후)
- 월 소개 20건 × 평균 연결료 2.5만 = 50만 원
- 스폰서 슬롯 3개 × 10만 = 30만 원
- (12개월+) Pro 구독 15사 × 7만 = 105만 원
- **월 ~185만 원** — 1인+AI 운영비 커버 수준. 진짜 성장은 구독 전환율과 리스팅 수에 비례. 광고(1단계)·거래소 리스팅료(3단계)는 별도.

## 5. 제품 반영 (이번 커밋)
- 제휴 매칭 페이지에 **"요금 안내"** 섹션: 현재 무료(베타) + 3층 요금 예고 + 3원칙(소개를 판다/정보는 안 판다/대금 미관여)
- DB 스키마: `plans`·`subscriptions`·`credit_ledger`·`charges`(연결료/노출/구독/리스팅료 청구 기록, 자금 미보유 원칙 그대로 — 세모플 명의 이용료만) + `proposals.introduced_at`(과금 트리거 시점 기록)
